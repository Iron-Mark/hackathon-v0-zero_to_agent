## Proof Standard

A platform is live-proven only when all of these are true:

- The required production environment variables are configured.
- The public webhook URL is registered inside the platform provider.
- A real external platform event reaches the deployed webhook.
- HireProof returns a formatted chat verdict.
- A saved report link is produced under `/audit/chat_...`.
- Evidence is captured in `docs/demo/` or recorded in `docs/demo/proof-archive.md`.

Local `/api/chat/hireproof` smoke tests prove the shared reply path, but they do not prove Discord, Telegram, or WhatsApp provider delivery.

## Shared Prerequisites

### 0. Run the Controlled Proof Check

The repo includes a sanitized proof checker for the parts we can verify without provider dashboards:

```powershell
npm run proof:chat-live
```

This checks production readiness, health, webhook readiness gates, and the shared ChatSDK reply path. It writes the latest snapshot to:

```text
docs/demo/live-chat-proof-check-latest.json
```

Use the strict mode only after all provider credentials and webhook registrations are complete:

```powershell
npm run proof:chat-live:strict
```

Strict mode writes its own latest snapshot:

```text
docs/demo/live-chat-proof-check-strict-latest.json
```

Expected current result now:

- Slack: `ready`
- Discord: `ready`
- Telegram: `ready`
- WhatsApp/Zernio: `credential-gated`
- Shared chat reply path: `200` with a saved `/audit/chat_...` report URL

The controlled command fails if production health, readiness shape, webhook metadata routes, or the shared reply path regress. The strict command additionally fails while any of Discord, Telegram, or WhatsApp/Zernio remain credential-gated; it does not replace screenshot/log capture for real event proof.

## Discord Proof

### Public Install Link

Use Discord's provided Add App flow unless a custom onboarding page is needed:

```text
https://discord.com/oauth2/authorize?client_id=1500240100804530336&scope=bot%20applications.commands&permissions=0
```

The `client_id` is public and safe to share. Treat this as the install handoff link, not as live-provider proof by itself.

### Slash Commands

HireProof supports these Discord application commands:

```text
/verify job_post:<suspicious job post text or supported LinkedIn job URL>
/help
```

For URLs, `/verify` can expand supported LinkedIn, public ATS, and public careers pages before auditing. If a job URL cannot be expanded, paste the visible job title, company, pay, location, and application process instead of relying on the URL alone.

Register or refresh commands from the repo after setting `DISCORD_BOT_TOKEN` and `DISCORD_APPLICATION_ID`:

```powershell
npm run discord:commands
```

For faster test-server registration, set `DISCORD_GUILD_ID` before running the command. Leave `DISCORD_GUILD_ID` unset for global registration.

### Required Production Variables

```text
DISCORD_BOT_TOKEN
DISCORD_PUBLIC_KEY
DISCORD_APPLICATION_ID
REDIS_URL
APP_BASE_URL=https://hireproof-sigma.vercel.app
```

### Steps

1. Open the Discord Developer Portal.
2. Confirm the HireProof application credentials are still present in Vercel production.
3. Confirm the interaction endpoint is registered:

```text
https://hireproof-sigma.vercel.app/api/webhooks/discord
```

4. Check readiness:

```powershell
Invoke-WebRequest -Uri "https://hireproof-sigma.vercel.app/api/integrations/proof" | Select-Object -ExpandProperty Content
```

5. Send a real Discord event to HireProof, preferably from a test server job channel.
6. Use this message:

```text
Can you verify this job? Remote frontend intern, PHP 80,000 per week, recruiter only wants Telegram contact and says no interview is needed.
```

7. Confirm HireProof replies with a high-risk verdict.
8. Open the returned `/audit/chat_...` report link.
9. Capture screenshots and logs.
10. Update `docs/demo/proof-archive.md` with the Discord proof details.

### Pass Criteria

- `/api/integrations/proof` reports Discord as `ready`.
- Discord sends a real event to production.
- HireProof replies inside Discord.
- The reply includes a report URL.
- Vercel logs show the Discord webhook request.

## Telegram Proof

### Required Production Variables

```text
TELEGRAM_BOT_TOKEN
TELEGRAM_WEBHOOK_SECRET_TOKEN
TELEGRAM_BOT_USERNAME
REDIS_URL
APP_BASE_URL=https://hireproof-sigma.vercel.app
```

### Steps

1. Open BotFather in Telegram.
2. Confirm the HireProof bot is `@HireProof_Bot`.
3. Confirm the production env vars are still present in Vercel production.
4. Confirm the production webhook with Telegram:

```powershell
$token = "<TELEGRAM_BOT_TOKEN>"
$secret = "<TELEGRAM_WEBHOOK_SECRET_TOKEN>"
$url = "https://hireproof-sigma.vercel.app/api/webhooks/telegram"
Invoke-WebRequest -Method Post -Uri "https://api.telegram.org/bot$token/setWebhook" -Body @{ url = $url; secret_token = $secret }
```

5. Check readiness:

```powershell
Invoke-WebRequest -Uri "https://hireproof-sigma.vercel.app/api/integrations/proof" | Select-Object -ExpandProperty Content
```

6. Send the bot a real Telegram message:

```text
Please check this offer: frontend intern, PHP 80,000 weekly, no interview, recruiter wants me to move to Telegram only.
```

7. Confirm HireProof replies with a high-risk verdict.
8. Open the returned `/audit/chat_...` report link.
9. Capture Telegram chat screenshots and Vercel webhook logs.
10. Update `docs/demo/proof-archive.md` with the Telegram proof details.

### Pass Criteria

- `/api/integrations/proof` reports Telegram as `ready`.
- Telegram accepts the webhook registration.
- A real Telegram message reaches production.
- HireProof replies inside Telegram.
- The reply includes a report URL.
- Vercel logs show the Telegram webhook request.

## WhatsApp/Zernio Proof

### Required Production Variables

```text
ZERNIO_API_KEY
ZERNIO_WEBHOOK_SECRET
ZERNIO_BOT_NAME=HireProof
REDIS_URL
APP_BASE_URL=https://hireproof-sigma.vercel.app
```

### Steps

1. Open the Zernio WhatsApp workspace or inbox.
2. Create or select the HireProof bot/integration.
3. Copy the API key into `ZERNIO_API_KEY`.
4. Copy or create the webhook secret into `ZERNIO_WEBHOOK_SECRET`.
5. Register the production webhook:

```text
https://hireproof-sigma.vercel.app/api/webhooks/zernio
```

6. Deploy or redeploy after setting env vars.
7. Check readiness:

```powershell
Invoke-WebRequest -Uri "https://hireproof-sigma.vercel.app/api/integrations/proof" | Select-Object -ExpandProperty Content
```

8. Send a real WhatsApp message through the connected Zernio inbox:

```text
Can HireProof check this? Remote job, huge weekly pay, recruiter refuses email and only wants messaging app contact.
```

9. Confirm HireProof replies with a high-risk or caution verdict.
10. Open the returned `/audit/chat_...` report link.
11. Capture WhatsApp/Zernio screenshots and Vercel webhook logs.
12. Update `docs/demo/proof-archive.md` with the WhatsApp/Zernio proof details.

### Pass Criteria

- `/api/integrations/proof` reports WhatsApp as `ready`.

- The reply includes a report URL.
